# 人生苦难重重

人生苦难重重

这是个伟大的真理，是世界上最伟大的真理之一。它的伟大之处在于，一旦我们领悟了这句话的真谛，就能从苦难中解脱出来，实现人生的超越。只要我们真正理解并接受了人生苦难重重的事实，那么我们就会释然，再也不会对人生的苦难耿耿于怀了。

出自《少有人走的路》

# 看书却记不住？

看群里的很多小伙伴都在说，书太厚很难啃，看了完全没记住，或者是尝试去看但是完全看不下去

  1. 首先，我们需要先承认这本书确实很厚且晦涩难懂，这是客观事实，这也并不是你一个人的问题，大家几乎都一样。
  2. 那么有没有好的办法解决呢？当然有，下面的学习方法是我在学习过程中总结出来的一套流程，希望对大家有些帮助。

# 有效学习方法 提升效率

# 你要懂得，慢即是快。

作为一名程序员 👩‍💻，在编写代码的过程中都要求使用面向对象的思想进行编程，这样代码可维护性强而且很高级，我们会非常鄙视“面向过程式”编程，但是在学习中我们要转变思想，没错，我们要面向过程式学习。你要的是在学习过程中学习，而不是事后再整理。

# 按部就班，才能闲庭信步、不慌不忙。

跟着班班的进度，过快过慢都不太好，毕竟大家都是上班族，人的有效学习时间是有限的。（不过不排除有些同学时间富裕）总而言之，按照一定的节奏来。跟着班班的进度自己就不用把时间花在如何安排上了。

以下是我总结的学习套路：

之所以分为 part1 和 part2 表示 这两部分可以分开来做，并不需要连续，但是要先 1 后 2

# part1

1. 知道自己今天和明天要干什么，做到心中有数，对进度的掌控感会使你更有信心

2. 看视频+记笔记

看视频，一边看一边记笔记，最好是直接记在书上，查看携带都方便。（老师讲课的速度完全可以跟的上记笔记，我基本上是 2 倍速看），虽然 5A 宝典的内容已经很详细了，但是老师在课程中举的例子，都是很能帮助我们记忆知识点的。

3. 快速回顾知识点

每次看完一个视频小节不要急着看下一个，暂停一下，合上书快速回顾一下老师讲了什么，看看自己能说出几点？包括老师举例什么好玩的例子，对应的知识点是什么？这儿也就是我所说的面向过程试学习，你的目的是在看视频的过程中学习到知识，而不是快速把视频看完，给自己一种交差的感觉。我们要尽量学会而不是看过

4. 默写知识点

看完整个章节的视频，合上书，在完全不看书的情况的下，尽可能的按照过程结构逻辑顺序，默写你所能写出来的全部知识点，比如讲了几个过程？过程有些？这里有什么图、矩阵吗？有什么好玩的例子吗？

重要的话说三遍 ：

默写 不要看书！！！

默写 不要看书！！！

默写 不要看书！！！

当你 5-10 分钟过去了觉得自己实在想不来了，再去查漏补缺！这个时候可谓是事半功倍了。首先呢，去查自己知道但模糊的知识点。然后再对比自己漏掉的知识点。 这样下来，一遍遍的加深印象。

就好比你听到了一首很熟悉的歌曲，但就是想不起歌的名字，这个时候太难受了，我就一定要知道这歌名叫啥！查到了之后一定印象深刻、恍然大悟。

学习亦是如此。

5. 做书上的课后题

注意，这个时候不要翻书了，凭记忆和直觉做，对就是对 错就是错，做的时候做上标记，对于自己犹豫的题目做上记号。防止侥幸做对了后面再错。

# part2

6. 做习惯养成打卡

这个很重要

  • 第一点：检查你在课堂上学习的效果。
  • 第二点：这里面的题目全部是 PMBOK 书中的原话，可以按照题目先把错的题目从书上找到答案并标记，然后把对的依据找出来。在这个过程中其实你已经 带着目的去看书了，而不是为了看一遍。再加上 part1 部分已经对整体结构有了一定的了解。所以应该不会那么难。

7. 读 PMBOK

做云课堂上的课后习题，这个也可以看完视频后做（不过更推荐放到这里，因为有的题目和书上的题目一样，隔开一些时间可以检验自己到底记住没，而且刚看完视频就做题，太趁热打铁了，可能忘得快）

然后看全部讲解，再次去 PMBOK 中查找原文一一对应，加深印象。

8. 读 PMBOK

此时再看 PMBOK，相信你大概会有一些看红烧肉不是红烧肉的感觉了。
